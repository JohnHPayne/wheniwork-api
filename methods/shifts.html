Shifts provide the basis for scheduling. Many other objects, including Locations, Positions, Sites and Users all link through Shifts.

Shift Object

Key Description
id Unique identifier for the shift
user_id User id
location_id Location ID
position_id Position ID
site_id Site ID

This method allows you to search for shifts. See the parameters below for all the options.

HTTP Request

GET /2/shifts

Parameters

Key Description
start_time Start time for a search window
end_time End time for a search window
user_id Only show shifts for a user, or multiple (e.g. 1,5,3)
location_id Only show shifts for a location, or multiple.

Get Existing Shift

curl /shifts/1 \
 -H "W-Token: "

{
  "shift": {
    "id": 1,
    "user_id": 1,
    "location_id": 1,
    "start_time": "Wed, 06 Mar 2014 08:00:00",
    "end_time": "Wed, 06 Mar 2014 13:30:00",
    ...
    "notes": "Come in early today"
  }
}

HTTP Request

GET /2/shifts/{id}

Parameters

Key Description
id ID of the shift requested.

Create/Update Shift

Create HTTP Request

POST /2/shifts/

Update HTTP Requests

PUT /2/shifts/{id}

Post/Put Body

Key Description
id ID of the shift requested.

Delete Existing Shift

curl -X DELETE /shifts/1 \
 -H "W-Token: "

{
  "success": true
}

HTTP Request

DELETE /2/shifts/{id}

Parameters

Key Description
id ID of the shift requested.
ids Optional query parameter that can be used to delete multiple shifts. ?ids=5,2,34
